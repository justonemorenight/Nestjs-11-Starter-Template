## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ pnpm install
```

## Compile and run the project

```bash
# development
$ pnpm run start

# watch mode
$ pnpm run start:dev

# production mode
$ pnpm run start:prod
```

## Run tests

```bash
# unit tests
$ pnpm run test

# e2e tests
$ pnpm run test:e2e

# test coverage
$ pnpm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ pnpm install -g mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

## NestJS 11 Features Implementation

This project implements several new features from NestJS 11:

### JSON Logger
The application uses the enhanced ConsoleLogger with JSON formatting for better log parsing in containerized environments:

```typescript
const app = await NestFactory.create(AppModule, {
  logger: new ConsoleLogger({
    json: true,
    // Enable colors in development environment
    colors: process.env.NODE_ENV === 'development',
  }),
});
```

### Graceful Shutdown
Implemented the new termination lifecycle hooks order for better application shutdown:

```typescript
// In main.ts
app.enableShutdownHooks();
```

The hooks execution order in NestJS 11:
1. onModuleDestroy()
2. beforeApplicationShutdown()
3. onApplicationShutdown()

### Express v5 Support
The project uses Express v5 with its updated path route matching algorithm and new routing syntax.

#### Path Route Matching Changes
Express v5 introduces several important changes to route matching:

1. Named Wildcards
```typescript
// Old syntax (Express 4)
app.get('/files/*', handler);

// New syntax (Express 5)
app.get('/files/*splat', handler);
// or
app.get('/files/{*wildcard}', handler);
```

2. Optional Parameters
```typescript
// Old syntax (Express 4)
app.get('/files/:name?', handler);

// New syntax (Express 5)
app.get('/files/{:name}', handler);
```

3. File Extensions
```typescript
// Old syntax (Express 4)
app.get('/:file.:ext?', handler);

// New syntax (Express 5)
app.get('/:file{.:ext}', handler);
```

4. Reserved Characters
The following characters are now reserved: `()[]?+!`
Use backslash to escape them if needed: `\(`, `\)`, etc.

5. Parameter Names
Now support valid JavaScript identifiers or quoted names:
```typescript
app.get('/users/:"user-id"', handler);
```

These changes ensure more consistent and predictable route matching behavior across the application.
